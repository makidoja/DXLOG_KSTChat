DXLog KST Chat Bridge with AirScout v2.3.3

Version 2.3.3 is a major feature and usability update from v2.1.3, adding improved AirScout opportunity displays, selectable worked-band columns, station watchlists, filtering, alerts, map enhancements and more reliable right-click handling.

Changes since v2.1.3
AirScout opportunity display

The AS column now uses clear graphical indicators:

Green marker — suitable aircraft available now
Orange marker with time — aircraft opportunity approaching
Dash — AirScout replied but found no suitable aircraft
Blank — station has not yet been scanned

Aircraft shown on the map use the same green and orange status colours.

The automatic AirScout scan continues to run in the background and prioritises watched stations.

Selectable worked-band columns

The previous combined Worked text has been replaced by individual selectable band columns.

Available bands include:

50 MHz
70 MHz
144 MHz
432 MHz
1296 MHz
2320 MHz
3400 MHz
5760 MHz
10 GHz
24 GHz
47 GHz
76 GHz

A green tick indicates that the station has been worked on that band in the current DXLog log.

The bands displayed can be selected in Setup.

Opportunity filtering and sorting

A new AS filter allows the station list to show:

All stations
NOW
Within 5 minutes
Within 10 minutes
Within 20 minutes

Automatic AS sorting can place immediate and approaching opportunities at the top of the list while retaining the currently selected station.

Station watchlist

Stations can now be added to a persistent watchlist using the right-click menu.

Watched stations:

show a star indicator
are scanned first by AirScout
remain visible through distance and AS filters
can trigger AirScout opportunity alerts
AirScout alerts

Optional alerts can be generated when a watched or visible station enters a selected opportunity window.

Alert options include:

configurable warning time
notification when an opportunity becomes NOW
optional sound
optional Windows balloon notification

Alerts are rate-limited to avoid repeated notifications for the same opportunity.

Improved station tooltips

The AS tooltip now includes more detailed information where available:

aircraft callsign
opportunity time
category
potential
path intersection distance
altitude
speed
track

Station tooltips also continue to show last activity and worked-band information.

Map improvements

The map now includes:

green aircraft for NOW opportunities
orange aircraft for approaching opportunities
optional aircraft trails
improved label priority and collision avoidance
watched-station label priority
persistent selected-station path trace
preservation of aircraft positions during temporary feed errors
retention of the last good aircraft data when an update is empty or fails

The selected station trace and aircraft overlay are now stored separately, preventing aircraft refreshes from clearing the path.

New macro tokens

The following macro tokens have been added:

{LOC}
{MYLOC}
{QTF}
{QRB}
{AS}
{AIRCRAFT}
{ASMIN}

These allow macros to include the selected station locator, bearing, distance and current AirScout opportunity details.

Performance diagnostics

Right-clicking the bridge status line opens a performance summary showing information such as:

number of KST users
visible users
AirScout results
AirScout reply time
aircraft feed time
KST refresh time
map render time

This is intended to help identify any performance issues during contest operation.

Right-click menu improvements

Context menus are now separated by area:

Right-clicking a station shows KST Bridge station commands.
Right-clicking a message shows callsign-related commands.
Right-clicking M1–M4 edits that specific macro.
Right-clicking the status line opens performance diagnostics.
Right-clicking a station on the map shows the bridge callsign menu.
The DXLog colour/options menu is available only from the green KST Chat Bridge title bar.

Version 2.3.3 also adds a context-menu shield to stop DXLog’s underlying log-entry menu appearing at the same time as the KST Bridge station menu.

Reliability fixes
Selected map traces no longer disappear during aircraft updates.
Aircraft no longer all vanish during a temporary empty or failed AirScout HTTP update.
The last good aircraft list is retained until valid replacement data is received.
The KST station and message lists now consume their own right-click events rather than passing them through to DXLog.
Map and AirScout state remain independent during refreshes.
Upgrade notes

Version 2.3.3 retains the existing v2.1.3 KST, AirScout, room, colour and macro configuration where possible.

After upgrading, open Setup and select which worked-band columns you want displayed. New installations default to showing 144 MHz and 432 MHz.

AirScout remains the aircraft-position and path-calculation engine. The normal connection is:

Aircraft feed → AirScout → UDP 9872 / HTTP 9880 → KST Bridge

Build the project using Release | x86 and replace the previous bridge DLL in the DXLog installation.
